# Feature Specification: 開発ツール非表示 & APIエラー通知UI

**Feature Branch**: `002-devtools-error-ui`  
**Created**: 2026-05-11  
**Status**: Draft  
**Input**: User description: "Nextの開発ツールみたいなのがずっと出てるのが気になるから無効にしてほしい。また、APIリクエストに失敗した時にそれがわかるようなUIを追加したい"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 開発ツールインジケーターを非表示にする (Priority: P1)

ユーザーはアプリを使用中に画面に常駐する開発ツールのインジケーター（Next.js devtools badge）が邪魔に感じている。本番相当の体験を得るため、このUIを非表示にしたい。

**Why this priority**: 見た目の問題で常時発生しており、ユーザー体験を直接損なっている。実装コストも低いため最優先。

**Independent Test**: アプリを起動して画面上に開発ツールのバッジ・インジケーターが表示されないことを確認するだけで価値が確認できる。

**Acceptance Scenarios**:

1. **Given** アプリを開いた状態で, **When** 画面を確認する, **Then** Next.js の開発ツールインジケーターが表示されていない
2. **Given** 任意のページを表示している, **When** 画面のどの位置も確認する, **Then** 開発ツールのバッジや浮遊するUI要素が存在しない

---

### User Story 2 - APIリクエスト失敗時にエラーを通知する (Priority: P2)

ユーザーが在庫の登録・更新・削除や買い物リストの操作を行った際にAPIリクエストが失敗した場合、何も起きないかのように見えてしまう。失敗したことをユーザーが認識できるよう、画面上でフィードバックを提供する。

**Why this priority**: ユーザーが操作の成否を把握できないと、データの不整合や誤操作に繋がる。P1の後に追加することで独立してテスト可能。

**Independent Test**: APIリクエストが意図的に失敗する状態（例: ネットワーク切断）でアイテムの登録や数量変更を試みたとき、エラーメッセージが画面に表示されることで確認できる。

**Acceptance Scenarios**:

1. **Given** 在庫の登録・更新・削除操作を行った, **When** APIリクエストが失敗する, **Then** 画面上に「失敗しました」などのエラーメッセージが表示される
2. **Given** 買い物リストの操作（追加・チェック・削除）を行った, **When** APIリクエストが失敗する, **Then** 画面上にエラーメッセージが表示される
3. **Given** エラーメッセージが表示された, **When** 一定時間が経過する, **Then** エラーメッセージが自動的に消える
4. **Given** エラーメッセージが表示されている, **When** ユーザーが手動で閉じる, **Then** エラーメッセージが即座に消える
5. **Given** 複数の操作が連続して失敗した, **When** エラーが複数発生する, **Then** それぞれのエラーが適切に表示される（上書きや消失なし）

---

### Edge Cases

- ネットワーク断絶とサーバーエラーは同じエラー表示で統一する（詳細を出しすぎない）
- エラー表示中に別のエラーが発生した場合、最新のエラーも追加表示する
- 認証切れなど特定のエラーに対して別の動作（リダイレクトなど）が発生する場合も、エラーUIを合わせて表示する

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: アプリのいかなる画面においても、Next.js の開発ツールインジケーターを表示してはならない
- **FR-002**: 在庫API（登録・更新・削除）のリクエストが失敗した場合、画面上にエラー通知を表示しなければならない
- **FR-003**: 買い物リストAPI（追加・チェック・削除）のリクエストが失敗した場合、画面上にエラー通知を表示しなければならない
- **FR-004**: エラー通知は操作失敗から3秒以内に表示されなければならない
- **FR-005**: エラー通知は一定時間（5秒程度）後に自動消滅しなければならない
- **FR-006**: エラー通知はユーザーが手動で閉じられなければならない
- **FR-007**: エラー通知はメインコンテンツの操作を妨げない位置（画面の端）に表示されなければならない
- **FR-008**: 複数のエラーが連続して発生した場合、それぞれが表示されなければならない

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: アプリの全画面で開発ツールインジケーターが視覚的に存在しない
- **SC-002**: APIが失敗した全ての操作に対して、3秒以内にエラー通知が表示される
- **SC-003**: エラー通知は5秒以内に自動消滅する
- **SC-004**: エラー通知の表示中もユーザーが画面の主要コンテンツを操作できる

## Assumptions

- エラー通知のデザインはアプリの既存スタイル（黒・白・グレー基調）に合わせる
- エラーメッセージは日本語で表示する
- 成功時の通知（トースト）は本スコープ外とし、エラー通知のみ実装する
- 開発ツールの非表示は全環境（開発・本番）で適用する
- エラー通知の対象はユーザー起因のミューテーション操作（登録・更新・削除）とし、データ取得（一覧表示）の失敗は対象外とする
