# Feature Specification: 在庫管理アプリ（StockMate）

**Feature Branch**: `001-inventory-management`  
**Created**: 2026-05-10  
**Status**: Draft  
**Input**: User description: 食品・消耗品などの在庫管理を行うWebアプリ

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 在庫を登録する (Priority: P1)

ユーザーは購入した食品や消耗品を在庫として登録する。品目・数量・カテゴリは必須で入力し、品目欄には既存の在庫品目からサジェストが表示される。購入日はデフォルトで今日の日付が入っており、賞味期限・消費期限や常備品フラグも任意で設定できる。

**Why this priority**: 在庫を登録できなければアプリ全体が成立しないため、最優先。

**Independent Test**: 登録フォームから1件在庫を追加し、在庫一覧に表示されることを確認するだけで価値が確認できる。

**Acceptance Scenarios**:

1. **Given** 在庫登録画面を開いている, **When** 品目・数量・カテゴリを入力して登録ボタンを押す, **Then** 在庫一覧に新しいアイテムが追加される
2. **Given** 品目欄に文字を入力している, **When** 既存の在庫品目に一致する文字列を入力する, **Then** 候補がドロップダウンで表示される
3. **Given** 在庫登録画面を開く, **When** 購入日欄を確認する, **Then** 今日の日付がデフォルトで入っている
4. **Given** 必須項目（品目・数量・カテゴリ）を未入力, **When** 登録ボタンを押す, **Then** バリデーションエラーが表示され登録されない
5. **Given** 数量欄に整数以外（小数・マイナス・文字など）を入力, **When** 登録しようとする, **Then** バリデーションエラーが表示される

---

### User Story 2 - 在庫一覧を確認・管理する (Priority: P2)

ユーザーは在庫一覧画面で全ての在庫を確認し、インクリメンタルサーチで素早く目的のアイテムを絞り込む。在庫数は一覧画面上で＋/−ボタンを押すだけで即座に調整でき、スマートフォンではスワイプ操作で「使い切り（削除）」や「買い物リストへ追加」ができる。期限が近いアイテムは視覚的に強調表示される。

**Why this priority**: 毎日の在庫確認・調整が主要ユースケースであり、一覧画面は使用頻度が最も高い。

**Independent Test**: 複数件の在庫データがある状態でインクリメンタルサーチと数量±調整が動作することを確認できる。

**Acceptance Scenarios**:

1. **Given** 複数の在庫が登録されている, **When** 検索窓に文字を1文字入力する, **Then** 入力した文字を含むアイテムだけにリストが絞り込まれる
2. **Given** 在庫一覧を表示している, **When** アイテムの＋ボタンを押す, **Then** そのアイテムの在庫数が1増加する
3. **Given** 在庫一覧を表示している, **When** アイテムの−ボタンを押す, **Then** そのアイテムの在庫数が1減少する（最小値は0）
4. **Given** スマートフォンで在庫一覧を表示している, **When** アイテムを右スワイプする, **Then** 「使い切り」確認が表示されOKすると削除される
5. **Given** スマートフォンで在庫一覧を表示している, **When** アイテムを左スワイプする, **Then** そのアイテムが買い物リストに追加される
6. **Given** 期限まで7日以内のアイテムが存在する, **When** 在庫一覧を表示する, **Then** そのアイテムの背景が薄い赤色で表示される
7. **Given** データを取得中である, **When** 在庫一覧画面を表示する, **Then** アイテムカードのスケルトンが表示される

---

### User Story 3 - 買い物リストを管理する (Priority: P3)

ユーザーは買い物リスト画面でチェックリスト形式の買い物リストを参照し、購入済みのアイテムにチェックを入れながら買い物を進める。在庫一覧でのスワイプ操作や常備品在庫切れによる自動追加でリストが更新される。リストへの手動追加・削除も可能。

**Why this priority**: 在庫登録・一覧管理が完成した後に意味をなす機能であり、優先度はP3。

**Independent Test**: 手動でアイテムを買い物リストに追加し、チェックを入れて削除する一連の操作が機能することで確認できる。

**Acceptance Scenarios**:

1. **Given** 買い物リストにアイテムが存在する, **When** リスト画面を開く, **Then** 未購入アイテムがチェックボックス付きで一覧表示される
2. **Given** 買い物リストのアイテムにチェックを入れる, **When** チェックする, **Then** そのアイテムが購入済み状態（視覚的に区別）になる
3. **Given** 常備品フラグが設定されたアイテムの在庫が0になる, **When** 在庫数が0に更新される, **Then** 自動的に買い物リストに追加される（重複しない）
4. **Given** 在庫一覧でアイテムを左スワイプする, **When** 操作が完了する, **Then** そのアイテムが買い物リストに追加される（重複しない）
5. **Given** 買い物リストを表示している, **When** アイテムの削除ボタンを押す, **Then** そのアイテムがリストから削除される

---

### User Story 4 - ホーム画面にアプリを追加する (Priority: P4)

ユーザーはスマートフォンのブラウザからアプリをホーム画面に追加し、ネイティブアプリのように使用する。

**Why this priority**: コア機能ではないが、モバイルでの日常利用に直結するUX向上機能。

**Independent Test**: ブラウザの「ホーム画面に追加」プロンプトが表示され、追加後にアイコンからアプリが起動することで確認できる。

**Acceptance Scenarios**:

1. **Given** スマートフォンのブラウザでアプリを開く, **When** ブラウザのメニューを開く, **Then** 「ホーム画面に追加」オプションが表示される
2. **Given** アプリをホーム画面に追加済み, **When** ホーム画面のアイコンをタップする, **Then** アプリがスタンドアロンモードで起動する

---

### Edge Cases

- 在庫数が0のアイテムを−ボタンで減らそうとした場合は0のまま維持する
- 同一品目が複数件登録されている場合、それぞれ別の在庫として管理する
- 常備品アイテムが既に買い物リストに存在する状態で在庫が0になった場合、重複追加しない
- 在庫一覧でのスワイプ中に誤操作を防ぐため、削除前に確認ダイアログを表示する
- 期限日未設定のアイテムは期限警告を表示しない
- 検索窓に入力がない場合は全件表示する

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは品目・数量（正の整数）・カテゴリを必須入力として在庫を登録できなければならない
- **FR-002**: 品目入力時に既存の在庫品目名からインクリメンタルサーチによるサジェストを表示しなければならない
- **FR-003**: 購入日のデフォルト値は当日の日付でなければならない
- **FR-004**: ユーザーは賞味期限・消費期限を任意で登録できなければならない
- **FR-005**: ユーザーは常備品フラグを在庫アイテムに設定できなければならない
- **FR-006**: 在庫一覧は品目名によるインクリメンタルサーチで絞り込みができなければならない
- **FR-007**: 在庫一覧上で＋/−ボタンにより在庫数をインラインで調整できなければならない（最小値は0）
- **FR-008**: スマートフォンでの右スワイプにより在庫アイテムを削除できなければならない（確認ダイアログあり）
- **FR-009**: スマートフォンでの左スワイプにより在庫アイテムを買い物リストへ追加できなければならない
- **FR-010**: 賞味期限・消費期限が7日以内のアイテムは視覚的に強調表示されなければならない
- **FR-011**: データ取得中はスケルトン表示を行わなければならない
- **FR-012**: 買い物リストをチェックボックス付きのリスト形式で表示・管理できなければならない
- **FR-013**: 常備品フラグが設定されたアイテムの在庫数が0になった場合、自動的に買い物リストへ追加されなければならない（重複なし）
- **FR-014**: 在庫一覧からの左スワイプで追加したアイテムは買い物リストへ追加されなければならない（重複なし）
- **FR-015**: アプリはPWAとしてデバイスのホーム画面に追加できなければならない
- **FR-016**: カテゴリは「調味料」「冷凍食品」「日用品」などの選択肢から選ぶ形式でなければならない
- **FR-017**: ユーザーはメールアドレスとパスワードでアカウントを作成・ログインできなければならない
- **FR-018**: 在庫データおよび買い物リストは同一アカウントにログインしたユーザー間で共有されなければならない
- **FR-019**: ログインしていないユーザーは在庫・買い物リストにアクセスできてはならない

### Key Entities *(include if feature involves data)*

- **在庫アイテム（InventoryItem）**: 品目名、数量（正の整数）、カテゴリ、購入日、期限日（任意）、常備品フラグを持つ
- **カテゴリ（Category）**: 在庫アイテムを分類するラベル。初期値として「調味料」「冷凍食品」「日用品」等を提供
- **買い物リストアイテム（ShoppingListItem）**: 買い物リストに追加されたアイテム。品目名と購入済みフラグを持つ

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: ユーザーは在庫登録を1分以内に完了できる
- **SC-002**: 在庫一覧での検索は入力から500ミリ秒以内に結果を表示する
- **SC-003**: 在庫数の±調整はタップから1秒以内に画面に反映される
- **SC-004**: 常備品の在庫が0になってから5秒以内に買い物リストへ自動追加される
- **SC-005**: PWAとしてホーム画面に追加したアプリは3秒以内に起動する
- **SC-006**: モバイル端末での主要操作（登録・一覧確認・数量調整）が片手で完結する

## Assumptions

- 複数ユーザー（家族間共有）に対応する。ユーザー登録・ログイン機能を持ち、在庫データ・買い物リストは同一アカウント内で共有される
- 期限警告の閾値は7日前とする
- カテゴリは初期値として「調味料」「冷凍食品」「日用品」「飲料」「野菜・果物」「肉・魚」「お菓子」「掃除用品」「衛生用品」を提供する（カスタム追加は対象外）
- オフラインでの動作（PWA）はv1では対象外とし、ホーム画面への追加のみ対応する
- スワイプアクションはモバイルブラウザのみを想定し、デスクトップではボタンUIで代替する
- 削除した在庫アイテムの履歴保持はv1スコープ外とする
