# Feature Specification: スマート在庫同期・UI改善

**Feature Branch**: `005-smart-inventory-sync`  
**Created**: 2026-05-11  
**Status**: Draft  
**Input**: User description: "在庫管理一覧で-ボタンを押して常備品の在庫が0になったら買い物リストへ自動追加してトースト通知。買い物リストでチェックを付けたアイテム（在庫一覧から追加されたもの）は在庫数を自動で+1してトースト通知。全体フォントサイズを大きくしてスマホで見やすくする。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 常備品の在庫切れで買い物リストへ自動追加 (Priority: P1)

在庫一覧で「常備品」フラグが付いたアイテムの在庫数を「−」ボタンで減らして0になった瞬間に、そのアイテムが自動で買い物リストへ追加される。ユーザーは手動操作なしで補充が必要なことを把握できる。

**Why this priority**: 「常備品」の主目的は在庫切れ時の自動補充。これが機能しないと登録した意味がない。

**Independent Test**: 常備品として登録済みのアイテムの在庫を「−」ボタンで1→0にしたとき、買い物リストに自動追加され画面上部に通知が表示されることで確認できる。

**Acceptance Scenarios**:

1. **Given** 常備品フラグありのアイテムの在庫が1, **When** 「−」ボタンを押す, **Then** 在庫が0になり、アイテムが買い物リストへ追加され、「〇〇を買い物リストへ自動追加しました」という通知が画面上部に数秒表示されてフェードアウトする
2. **Given** 常備品フラグありのアイテムがすでに買い物リストに存在する, **When** 在庫が0になる, **Then** 重複追加はされず、通知も表示しない（または「すでに追加済み」旨の通知を表示する）
3. **Given** 常備品フラグなしのアイテムの在庫が1, **When** 「−」ボタンを押す, **Then** 在庫が0になるが買い物リストへの自動追加は行われない
4. **Given** 常備品フラグありのアイテムの在庫が2以上, **When** 「−」ボタンを押す, **Then** 在庫が減るが0にならないので自動追加は発生しない

---

### User Story 2 - 買い物リストのチェックで在庫を自動補充 (Priority: P2)

買い物リストにある「在庫一覧から追加されたアイテム」のチェックボックスをオンにすると、対応する在庫アイテムの在庫数が自動で+1される。買い物から帰ったあとの在庫登録作業が省略できる。

**Why this priority**: 買い物完了と在庫更新を同時に行えるため、二重入力の手間を削減する。

**Independent Test**: 在庫一覧から買い物リストへ追加されたアイテムのチェックをオンにしたとき、在庫数が+1され通知が表示されることで確認できる。

**Acceptance Scenarios**:

1. **Given** 在庫一覧から追加されたアイテムが買い物リストに存在する, **When** チェックボックスをオンにする, **Then** 対応する在庫数が+1され、「〇〇の在庫数を+1しました」という通知が画面上部に数秒表示されてフェードアウトする
2. **Given** 手動で（在庫一覧以外から）追加されたアイテムが買い物リストに存在する, **When** チェックボックスをオンにする, **Then** 在庫の自動更新は行われない（通知も出ない）
3. **Given** チェックをオンにしてすでに在庫が更新済み, **When** チェックをオフにする, **Then** 在庫数は元に戻さない（減算しない）

---

### User Story 3 - フォントサイズ拡大でスマホ視認性向上 (Priority: P3)

アプリ全体のフォントサイズが現状より大きくなり、スマートフォンでの読みやすさが向上する。

**Why this priority**: UI改善であり機能には影響しないが、モバイルファーストを掲げるアプリとして重要。

**Independent Test**: スマートフォン（または同等の画面幅）でアプリを表示し、品目名・カテゴリ名・ボタンラベル等が読みやすいサイズで表示されることを目視確認できる。

**Acceptance Scenarios**:

1. **Given** スマートフォン（375px幅）でアプリを表示する, **When** 在庫一覧・買い物リスト・在庫登録画面を閲覧する, **Then** すべてのテキストが目視で読みやすいサイズで表示される
2. **Given** デスクトップブラウザでアプリを表示する, **When** 各画面を閲覧する, **Then** レイアウトが崩れず、文字サイズの変更が適切に反映されている

---

### Edge Cases

- 常備品アイテムの在庫が0の状態で「+」ボタンを押した後、再度「−」で0にした場合、買い物リストへの追加が重複しないか
- 買い物リストのチェックをオンにした直後に通信エラーが発生した場合、在庫の不整合が起きないか（ユーザーへのエラー通知）
- フォントサイズ変更により既存のボタンやアイコンが意図せず崩れないか

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 「常備品」フラグが有効なアイテムの在庫数が0になったとき、システムは自動でそのアイテムを買い物リストへ追加しなければならない
- **FR-002**: 自動追加が実行された場合、システムは画面上部に「〔アイテム名〕を買い物リストへ自動追加しました」という通知を数秒間表示し、自動的にフェードアウトしなければならない
- **FR-003**: 対象アイテムがすでに買い物リストに存在する場合、重複追加は行わないこと
- **FR-004**: 在庫一覧から追加されたアイテムに限り、買い物リストでチェックをオンにしたとき、対応する在庫数を+1しなければならない
- **FR-005**: 在庫の自動補充が実行された場合、システムは画面上部に「〔アイテム名〕の在庫数を自動で+1しました」という通知を数秒間表示し、自動的にフェードアウトしなければならない
- **FR-006**: チェックをオフにしても在庫数の自動減算は行わないこと
- **FR-007**: アプリ全体の本文・ラベル・見出しのフォントサイズを、スマートフォンで視認しやすいサイズへ引き上げること
- **FR-008**: フォントサイズ変更後も既存のレイアウト・デザインが崩れないこと

### Key Entities

- **在庫アイテム（InventoryItem）**: 品目名、カテゴリ、在庫数、常備品フラグを持つ
- **買い物リストアイテム（ShoppingListItem）**: 品目名、追加元（在庫一覧 or 手動）のフラグ、チェック状態を持つ
- **通知（Toast）**: 表示テキスト、表示時間（秒）、フェードアウト挙動を持つ

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 常備品アイテムの在庫が0になった場合、100%のケースで買い物リストへの自動追加と通知表示が実行される
- **SC-002**: 通知は操作後2秒以内に表示され、3〜5秒後に自動的に消える
- **SC-003**: 在庫一覧から追加された買い物リストアイテムをチェックしたとき、100%のケースで在庫数が+1される
- **SC-004**: スマートフォン（375px幅）で全画面のテキストが読みやすく表示され、テスト担当者の目視確認を通過する
- **SC-005**: フォントサイズ変更後も既存機能のレイアウトテストをすべて通過する

## Assumptions

- 「常備品」フラグはすでに在庫アイテムのデータ属性として存在する（または今回追加する）
- 買い物リストアイテムには「在庫一覧から追加されたか否か」を識別する属性が追加可能である
- 通知（トースト）の表示時間は3〜5秒とし、ユーザー設定は不要
- フォントサイズの具体的な数値は実装時に調整するが、スマートフォンで読みやすい水準を目標とする
- チェックをオフにした場合の在庫の自動減算は今回のスコープ外
