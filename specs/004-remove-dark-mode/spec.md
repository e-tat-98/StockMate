# Feature Specification: ダークモード切り替えロジック削除

**Feature Branch**: `004-remove-dark-mode`  
**Created**: 2026-05-11  
**Status**: Draft  
**Input**: User description: "端末のライトモード＆ダークモードによる表示切り替えロジックを削除したい"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - アプリが常にライトモードで表示される (Priority: P1)

ユーザーが端末をダークモードに設定していても、アプリは常にライトモード（白背景・黒文字）で表示される。

**Why this priority**: ダークモード対応のスタイルが不完全なため表示崩れが発生している。常時ライトモード固定にすることで全ユーザーに一貫した表示を保証できる。

**Independent Test**: 端末をダークモードにした状態でアプリを開き、すべての画面が白背景・黒文字で正常に表示されることを確認する。

**Acceptance Scenarios**:

1. **Given** 端末がダークモードに設定されている, **When** アプリを起動する, **Then** すべての画面がライトモードのデザインで表示される
2. **Given** 端末がライトモードに設定されている, **When** アプリを起動する, **Then** すべての画面がライトモードのデザインで表示される
3. **Given** アプリ使用中に端末のモードを切り替える, **When** 画面を操作する, **Then** アプリの表示が変化しない（ライトモードのまま）

---

### Edge Cases

- 端末のシステム設定が「自動（時刻で切替）」になっている場合でも、アプリ側は変化しない
- ブラウザの「ダークモード優先」設定がある場合でも、アプリ側は変化しない

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: アプリは端末・ブラウザのカラーモード設定（ライト／ダーク）に関わらず、常にライトモードのデザインを表示しなければならない
- **FR-002**: ダークモード用のスタイル定義（`dark:` プレフィックスのユーティリティクラス、`prefers-color-scheme: dark` のメディアクエリ等）を削除または無効化しなければならない
- **FR-003**: 削除後も既存のライトモードのデザイン・レイアウトはそのまま維持されなければならない

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 端末をダークモードに設定した状態でアプリのすべての画面を確認し、背景・文字・アイコン等の色が変化しないこと
- **SC-002**: ライトモード設定時と同一の見た目が維持されること
- **SC-003**: 既存機能（在庫一覧、買い物リスト、在庫登録）の表示・操作に問題がないこと

## Assumptions

- ダークモード対応を将来的に再導入する予定はない（削除で十分）
- 対象はWebアプリ全体（全ページ・全コンポーネント）
- ライトモードのデザインは現状のまま正しいものとして扱う
